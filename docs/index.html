CAPABILITIES

Everything a local SIEM should be.

🧠
Local LLM Analysis
Rule engine first, then high-value logs go to your model for context-aware threat detection, summaries, and suggested fixes. Excellent with Qwen2.5, Llama 3.1/3.3, Command-R.
🏠
Home Assistant Superpowers
Pulls your actual device registry. Matches IPs/MACs to lights, sensors, switches. Fires notifications, events, updates sensors. Forward all addon + core + supervisor logs via the syslog addon.
💓
Active Heartbeat Monitoring
Not just "is the port open?". Verifies the *right* website version, SSH config up to date, services healthy. Failed or stale heartbeats become first-class threats that go through the full analysis pipeline.
📅
Daily Briefing (Operator Companion)
Entertaining, grounded recap of the day/shift. Chat naturally: "why the spike at 3am?", "draft a fix script", "promote this to a standing monitor". Full history persisted.
🤖
Built-in AI Assistant
Ask about config, iBMi automation, monitors, anything. Self-documenting. Submit feature suggestions through the UI — admins review them in the same place.
🗺️
Offline + Multi-Source Geo
Free MaxMind GeoLite2 (city level) + optional paid providers merged with priority. Every threat gets location without leaking IPs to third parties.
🛠️
Guarded Remediation & Monitors
Huge library of prebuilt remediation scripts (Linux, Windows, Cisco, Ubiquiti, pfSense, IBM i, cert renewal, etc.). English → safe script generator. All actions require explicit enable + confirmation.
🔌
Broad Log Sources
Syslog UDP/TCP (real 514 or 5140), network gear, Linux rsyslog, Windows (NXLog etc), HA addons, Docker logs, custom apps. RFC3164 + 5424 + best-effort.
🛡️
Fast Deterministic Rules + MAC + Blacklists
Built-in high-value patterns for brute force, sudo abuse, crypto miners, kernel panics, etc. + custom patterns. MAC vendor lookup (IEEE + Wireshark manuf) + learned device behavior profiles.
PIPELINE

Simple. Powerful. Transparent.

01
Ingest
Syslog UDP/TCP listeners. Parse RFC3164/5424. Store in SQLite with retention.
02
Rule Engine
Fast deterministic patterns + severity scoring. Immediate threat records for matches.
03
LLM Analysis
Periodic batches of interesting logs sent to *your* local model. Structured JSON: summary, severity, recommended action, IOCs.
04
Enrich & Alert
Geo (offline), device intel (HA + MAC), heartbeats, blacklists. Webhooks + rich HA notifications.
05
Act (optional)
Human-in-the-loop confirmation before any remediation. Full audit trail. Promote good fixes to monitors.
Everything stays on your box. The only "AI" calls are to the LLM endpoint *you* configured (localhost:1234 by default).

5 MINUTES TO FIRST THREAT

Get started

RECOMMENDED Docker Compose (easiest) 
git clone https://github.com/AnythingIP/RocketLogAI.git
cd RocketLogAI
cp example-config.yaml config.yaml
# Edit config.yaml — at minimum set llm.base_url
#   LM Studio: http://host.docker.internal:1234/v1   (or your LAN IP)
#   Ollama:    http://host.docker.internal:11434/v1
docker compose up -d --build
# Open http://your-server:8787   (default login: admin / admin — change it!)
Or download the v1.3 installer tarball for one-click Linux/Windows.
Native (Python)
git clone https://github.com/AnythingIP/RocketLogAI.git
cd RocketLogAI
pip install -e ".[web]"
logsentinel example-config -o config.yaml
# edit llm.base_url in config.yaml
logsentinel run
# or for real syslog port 514: sudo logsentinel run
Also ships one-click installers for Linux (systemd) and Windows (NSSM / scheduled task).
Need a local LLM? LM Studio or Ollama — load a good model (Qwen2.5 14B+ recommended) and point RocketLogAI at it.
